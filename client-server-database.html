Client-Server Database

The Project required the use of:

C#
Pipes
File IO
Threads
Mutex
Multiple Console apps

Details

The client side of the application would prompt the user to input a desired function from, write to database, find, update, or delete from the database. If there was anything incorrect on the client side, ie. Spelling mistake, incorrect option, the mistake was corrected before submitting information to the server-side portion. The client was connected to the server via the use of a pipe. The Server-side portion employed the use of threads so that it could accept multiple clients to the server. It also used a mutex to be able to effectively limit the usage of the flat file as multiple people could be requesting access to the database. The server-side portion then parsed the input string and would add, update, find or delete options as the user requested.
